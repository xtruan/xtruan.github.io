Chart Express for Android

 

This app uses an HTTP GET to receive an image from Googe's Chart API. It formats a URL with input data and some options in order to generate the desired chart for the user.

The major tasks in designing this application were creating a working HTTP client that would download a binary image, and then making said HTTP client run in the background so as not to block the GUI thread.

I created a simple HTTP client which is able to process and return the results of an HTTP GET from a given URL. This information is converted to a Drawable for display within the application.

To run the task in the background, I utilized the AsyncTask android library which allows a process to be assigned a thread from Android's thread pool as to not block the GUI thread. I believe this is the best way to accomplish this task, as creating a new thread to execute the HTTP client could lead to inefficiency from having too many threads operating concurrently.

The Drawable is converted to a Bitmap so it can be saved/sent as a JPEG. The chart.jpg file is stored in the DCIM folder on an Android device's SD card. The chart.jpg can be sent via the ACTION_SEND intent, which allows it to be sent by any application on the device that can handle an image (such as email or mms messenger).
